# Slide Agent Skill

テキスト入力からコンサル品質のPowerPointスライドを自動生成。

**核心原則: 文章が先、デザインは後**

---

## 適用タイミング

以下の場合にこのSkillを使用:
- 「議事録からスライドを作って」
- 「キックオフ資料を作成して」
- 「この内容をPPTにまとめて」

---

## 生成フロー（4段階）

```
入力: 議事録・テキスト
    ↓
Stage 1: 情報抽出
    → 基本情報、ゴール、体制、スケジュール、アクション、課題
    ↓
Stage 2: アジェンダ決定
    → 表紙、目次、ゴール、体制図、スケジュール、etc.
    ↓
Stage 3: 文章設計【最重要】
    → タイトル（15文字以内）
    → リード文（30文字以内、結論を先に）
    → ボディ（詳細・根拠）
    ↓
Stage 4: 表現形式決定
    → ボディ構造に最適なオブジェクトを選択
    ↓
出力: PPTX
```

---

## Stage 3: 文章設計ルール

**デザインを考える前に、何を伝えるかを文章として確定させる**

### タイトル
| ルール | 例 |
|-------|-----|
| 15文字以内 | 「プロジェクトゴール」 |
| 名詞句で終わる | 「〜について」は避ける |
| 1スライド1トピック | 複数トピックを混ぜない |

### リード文
| ルール | 例 |
|-------|-----|
| 30文字以内 | 「業務効率20%向上を目指す」 |
| 結論を先に | So What?への回答 |
| 1文で完結 | 「〜について説明します」はNG |

### ボディ
| 情報構造 | 例 |
|---------|-----|
| 階層的 | ゴール → サブゴール |
| 対比的 | 顧客側 vs 自社側 |
| 時系列的 | フェーズ1→2→3 |
| 並列的 | メンバーリスト |
| 因果的 | 課題 → 対策 |

---

## Stage 4: 表現形式マッピング

| ボディ構造 | 推奨オブジェクト |
|-----------|-----------------|
| **階層的** | ラベルBOX + コンテンツBOX + コネクタ |
| **対比的** | 2カラム |
| **時系列的** | タイムライン / ガントチャート |
| **並列的** | 箇条書きリスト |
| **因果的** | フロー図 / 矢印付き |

## 品質基準（Evals）

### ノックアウトファクター（即不合格）

| 項目 | 条件 |
|-----|------|
| グリッド整列違反 | 同列要素が0.2cm以上ズレ |
| はみ出し | スライド境界をはみ出す |
| サイズ不統一 | 同種オブジェクトのサイズ差10%以上 |

**1つでも該当すれば不合格。**

### スコアリング（100点満点）

| カテゴリ | 配点 | 評価観点 |
|---------|------|---------|
| 文章 | 40点 | 伝えたいことが明確か |
| オブジェクト | 30点 | 文章構造に最適な表現か |
| レイアウト | 30点 | 情報の流れが自然か |

### ランク判定

| ランク | スコア |
|-------|--------|
| S | 90-100 |
| A | 80-89 |
| B | 70-79（パスライン） |
| C | 50-69 |
| D | 0-49 |

## XMLテンプレート

### 定数

```python
# スライドサイズ
SLIDE_WIDTH = 12192000   # 33.87cm
SLIDE_HEIGHT = 6858000   # 19.05cm

# カラーパレット
PRIMARY = "0055FF"       # 青
SECONDARY = "80AAFF"     # 薄青
TEXT_DARK = "434343"     # 濃いグレー
WHITE = "FFFFFF"

# フォント
FONT_FAMILY = "BIZ UDPGothic"
```

### 利用可能なオブジェクト

| オブジェクト | 用途 |
|-------------|------|
| タイトルテキストボックス | スライドタイトル |
| ラベルBOX | カテゴリラベル（塗りつぶし） |
| コンテンツBOX | 詳細内容（枠線のみ） |
| コネクタ（L字） | 要素間の関係線 |
| 下向き矢印 | 階層の流れ |
| まとめテキストボックス | 下部の結論 |
