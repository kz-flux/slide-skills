# Slide Pro - 統合スライド作成 Skill

slide-master（情報設計）+ slide-design（技術実装）の統合版。
**設計思考フロー**と**パターンの適切な活用**を両立する。

---

## 核心思想

### パターンは「思考の起点」であり「制約」ではない

✅ **正しい使い方**:
- パターンを起点に考え始める
- 内容に合わせて調整・組み合わせ・逸脱する
- 「なぜこの表現がベストか」を常に問う

❌ **間違った使い方**:
- パターンに内容を合わせる
- 思考停止でパターンを適用
- 「パターンにあるから」で終わる

---

## 設計思考フロー

### Step 1: 目的の明確化

**最初に問う3つの質問**:

| 問い | 考えること |
|-----|-----------|
| **誰に**見せるか？ | 経営層 / マネージャー / 現場 / 顧客 |
| **何を**判断させたいか？ | 承認 / 理解 / 行動 / 投資 |
| **見た後に**どう動いてほしいか？ | 具体的なNext Action |

### Step 2: 情報の構造化

**伝えたい情報の本質的な構造を特定する**:

| 構造タイプ | 特徴 | 判断ポイント |
|-----------|------|------------|
| **数値比較** | KPI、スコア | 「数字が主役」なら |
| **順序・優先度** | ランキング | 「順番が意味を持つ」なら |
| **時系列** | ロードマップ | 「時間の流れがある」なら |
| **対比** | Before/After | 「比較が本質」なら |
| **因果** | 課題→解決策 | 「原因と結果がある」なら |
| **階層** | ゴール分解 | 「親子関係がある」なら |
| **プロセス** | ステップ | 「手順がある」なら |
| **分類** | カテゴリ | 「グループ分けが本質」なら |

**複合構造も考える**:
- 数値比較 + 対比 → 自社スコア vs 業界平均
- 順序 + 因果 → 課題TOP3 + 各課題の解決策
- 時系列 + 数値 → ロードマップ + 各フェーズのKPI

### Step 3: パターンを起点に表現を検討

**基本パターンから始めて、調整を検討**:

| 構造 | 起点パターン | 調整の観点 |
|-----|-------------|-----------|
| 数値比較 | KPIカード×3 | 数が多い→4列 or テーブル、少ない→2列大きく |
| 順序 | ランクカード×3 | 5個以上→リスト、詳細多い→1枚ずつ |
| 時系列 | タイムライン横 | 並行タスク→ガント、イベント→マイルストーン |
| 対比 | 2カラム | 項目多い→テーブル、3つ比較→3列 |

### Step 4: 最適化

**以下の観点で調整**:

| 観点 | 問い |
|-----|-----|
| 情報量 | 入りきらない→削る or 分割する |
| 強調 | 一番伝えたいことが一番目立っているか？ |
| 受け手 | 経営層→シンプルに、実務者→詳細OK |
| 流れ | 視線の動きは自然か？ |

## パターンからの逸脱基準

**以下の場合はパターンを崩してOK（むしろ崩すべき）**:

| 状況 | 対応 |
|-----|-----|
| 情報量がパターンに収まらない | 分割 or レイアウト変更 |
| 強調したい要素がある | サイズ・色で差別化 |
| ストーリーの流れで変化が必要 | 意図的にパターンを変える |
| 受け手が特殊 | 受け手に合わせて調整 |

**逸脱時の判断基準**:
- 「なぜ逸脱するか」を説明できるか？
- 逸脱した結果、より伝わりやすくなるか？

## 最適化のための問い

チェックリストではなく、**問い**として使う。

### 情報設計
- 一番伝えたいことは何か？それが一番目立っているか？
- 不要な情報は削れているか？
- So What?（だから何？）に答えているか？
- 見た人がNext Actionを理解できるか？

### 視覚設計
- 視線の流れは自然か？（左上→右下）
- 強弱のメリハリはあるか？
- 同じ重要度のものは同じ見た目か？
- 色の意味は一貫しているか？

### 受け手視点
- 3秒で要点が伝わるか？
- 詳細を見たい人は見れるか？
- 質問されそうなことは先回りしているか？

## ワークフロー

```
1. 目的確認
   └→ 誰に・何を・どう動いてほしいか

2. 情報構造の特定
   └→ 構造タイプ（複合含む）を判断

3. パターン選択
   └→ 起点パターンを選び、調整を検討

4. デザインすり合わせ
   └→ 2-3パターン提示 → ユーザー承認

5. 実装
   └→ python-pptxでネイティブオブジェクト

6. 最適化
   └→ 「問い」で自己レビュー

7. 納品
```

## 技術ルール（slide-designより）

### 編集可能性
python-pptxでネイティブオブジェクト生成。HTML→画像変換は禁止。

### 角丸四角形は禁止
```python
# OK
MSO_SHAPE.RECTANGLE

# NG
MSO_SHAPE.ROUNDED_RECTANGLE
```

### カラーパレット
```python
class ProjectColors:
    PRIMARY = RGBColor(0, 85, 164)      # ダークブルー
    PRIMARY_LIGHT = RGBColor(102, 153, 255)
    ACCENT = RGBColor(0, 150, 136)      # ティール（1色のみ）
    TEXT_DARK = RGBColor(33, 33, 33)
    TEXT_LIGHT = RGBColor(117, 117, 117)
    BG_LIGHT = RGBColor(250, 250, 250)
    BORDER = RGBColor(224, 224, 224)
    WHITE = RGBColor(255, 255, 255)
    ALERT = RGBColor(229, 57, 53)
```

---

## 色使いルール

### 禁止: パステル多色
```python
# NG: 派手なパステルの組み合わせ
RGBColor(200, 230, 201)  # パステルグリーン
RGBColor(187, 222, 251)  # パステルブルー
```

### 推奨: モノトーン + アクセント1色

### 色の判断フロー
```
強調したい数字 → アクセント色OK
区別したいカテゴリ → グレー濃淡で対応
彩りが欲しい → NG（余計）
```
