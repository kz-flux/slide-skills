# Slide Master - 情報設計 Skill

コンサルファームレベルのスライド品質を実現するための情報設計ガイドライン。

**役割分担**:
- **このSkill**: 何を作るか（情報設計・論理構造）
- **slide-design**: どう作るか（python-pptx実装）
- **slide-pro**: 両方を統合 + 設計思考フロー

---

## 原則（柔軟に適用）

### 必須

| 原則 | チェック |
|------|---------|
| 1スライド1メッセージ | タイトル＝結論 |
| So What? | 示唆が明確 |
| 構造化 | テキスト羅列NG |

### 推奨

| 原則 | チェック |
|------|---------|
| ピラミッド構造 | 結論→根拠→詳細 |
| 3の法則 | 項目3-5つ |
| MECE | 漏れなく重複なく |
| アクションドリブン | Next Actionあり |

### 状況次第

| 原則 | 適用場面 |
|------|---------|
| 余白の美学 | 情報量が多い場合は詰めてもOK |
| 視線誘導 | 複雑な図解の場合 |
| コントラスト | 強調したい数字がある場合 |

## 情報構造の分類

**スライドを設計する前に、伝えたい情報の構造を特定する。**

### 基本構造タイプ

| 構造タイプ | 特徴 | 例 |
|-----------|------|-----|
| **数値比較** | KPI、スコア、指標の提示 | 売上、成長率、スコア |
| **順序・優先度** | ランキング、重要度順 | 課題TOP3、優先施策 |
| **時系列** | 過去→現在→未来の流れ | ロードマップ、沿革 |
| **対比** | A vs B、Before/After | 現状と理想、競合比較 |
| **分類** | カテゴリ分け | 部門別、種類別 |
| **因果** | 原因→結果、課題→解決策 | 問題分析、提案 |
| **階層** | 上位→下位、全体→詳細 | 組織図、ゴール分解 |
| **プロセス** | ステップ、フロー | 業務フロー、手順 |

### 複合構造

実際のスライドは複数の構造が組み合わさることが多い:

| 組み合わせ | 例 |
|-----------|-----|
| 数値比較 + 対比 | 自社スコア vs 業界平均 |
| 順序 + 因果 | 課題TOP3 + 各課題の解決策 |
| 時系列 + 数値 | ロードマップ + 各フェーズのKPI |
| 分類 + 階層 | 3つの柱 + 各柱の詳細施策 |

## 情報構造 → スライドパターン対応表

**パターンは「思考の起点」であり「制約」ではない。**

### 基本パターン

| 構造タイプ | 起点パターン | 派生・組み合わせ |
|-----------|-------------|-----------------|
| **数値比較** | KPIカード×3 | 2列/4列、グラフ併用、ゲージ |
| **順序・優先度** | ランクカード×3 | バッジ付き、バー表示、マトリクス |
| **時系列** | タイムライン横 | 矢羽根、ガント、縦フロー |
| **対比** | 2カラム | Before/After、テーブル比較 |
| **分類** | カテゴリカード | タブ形式、アイコン付き |
| **因果** | 矢印フロー | 課題→解決策カード、ロジックツリー |
| **階層** | ピラミッド | 組織図、ツリー図 |
| **プロセス** | ステップ横並び | 番号付きフロー、サークル |

### パターン選択の判断軸

| 判断軸 | 考慮点 |
|-------|-------|
| 情報量 | 多い→テーブル、少ない→カード |
| 強調度 | 高い→大きく単独、低い→リスト化 |
| 比較性 | 比較重要→並列配置、単独説明→順次 |
| 受け手 | 経営層→シンプル、実務者→詳細OK |

## スライドパターン詳細

### 1. エグゼクティブサマリー

**用途**: 1枚で全体像を把握させる

```
[ヘッダー: タイトル + 日付]
[キーメッセージボックス: 結論1文]
[KPIカード×3: 主要指標]
[課題カード×3 or 箇条書き: 要点]
[ROIパネル or Next Action]
```

**派生**:
- KPI重視版: カード大きめ、詳細少なめ
- 詳細版: カード小さめ、テーブル追加
- アクション重視版: 下半分をNext Stepsに

### 2. 数値・KPI表示

**基本**: KPIカード×3

```
[タイトル]
[キーメッセージ]
[カード: ラベル / 大きな数字 / 補足]×3
```

**派生**:
- 2列版: 主要KPI2つを大きく
- 4列版: 指標多めだが個別の重要度低め
- グラフ併用版: カード + 推移グラフ
- 比較版: 自社 / 業界平均 / 目標 を並列

### 3. 課題・優先度表示

**基本**: ランクカード×3

```
[タイトル]
[キーメッセージ]
[カード: ランクバッジ / タイトル / 説明 / 推奨アクション]×3
```

**派生**:
- マトリクス版: 影響度×緊急度の4象限
- リスト版: 5個以上ある場合
- 詳細版: 1課題1スライドで深掘り

### 4. ロードマップ・時系列

**基本**: フェーズ矢羽根×3

```
[タイトル]
[キーメッセージ]
[Phase1 | Phase2 | Phase3] 矢羽根
[各フェーズの詳細: 期間、アクション、KPI]
```

**派生**:
- ガントチャート版: 複数ワークストリーム
- マイルストーン版: 主要イベントをポイント表示
- 縦フロー版: ステップが多い場合

### 5. 比較・対比

**基本**: 2カラム

```
[タイトル]
[キーメッセージ]
[左: As-Is / Before] [右: To-Be / After]
```

**派生**:
- テーブル版: 項目×選択肢のマトリクス
- 3列版: A案 / B案 / C案
- ビジュアル版: アイコン・図解で対比

## 色使いルール

### 禁止: パステル多色

```python
# NG: 派手なパステルの組み合わせ
RGBColor(200, 230, 201)  # パステルグリーン
RGBColor(187, 222, 251)  # パステルブルー
RGBColor(225, 190, 231)  # パステルパープル
```

### 推奨: モノトーン + アクセント1色

```python
# OK: グレー系 + メイン + アクセント1色
BG_LIGHT = RGBColor(250, 250, 250)
GRAY = RGBColor(235, 235, 235)
PRIMARY = RGBColor(0, 85, 164)    # ダークブルー
ACCENT = RGBColor(0, 150, 136)    # ティール（1色のみ）
```

### 色の意味づけ

| 用途 | 色 |
|-----|-----|
| 最重要・1位 | PRIMARY |
| 強調・警告 | ACCENT or 赤系 |
| 2位以降 | PRIMARY の薄い版 |
| 背景・区切り | グレー系 |
