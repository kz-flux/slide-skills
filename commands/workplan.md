# WBS & ワークプラン作成コマンド

**言語設定**: 全ての出力は日本語で行ってください。

---

## 読み込むSkills

**YOU MUST**: 以下のSkillsを必ず読み込んでから作業を開始してください。

```
@~/.claude/skills/wbs-excel/SKILL.md
@~/.claude/skills/workplan-slide/SKILL.md
```

---

## 概要

このコマンドは以下の2つを作成します：

| 成果物 | ツール | 用途 |
|--------|--------|------|
| **WBS（Excel）** | openpyxl | 詳細タスク管理・日程計算 |
| **ワークプラン（スライド）** | python-pptx | キックオフ等での説明用 |

---

## 実行フロー

### Phase 1: ヒアリング（最大3ラリー）

```
確認させてください：

1. プロジェクト名
2. 期間（開始日〜最終報告日）
3. 主要フェーズ（例: 要件定義、開発、テスト、納品）
4. 担当者の色分け（例: 顧客=オレンジ、自社=ブルー）

不明点は仮説で埋めますので、分かる範囲で大丈夫です。
```

### Phase 2: WBS作成（Excel）

**Skill参照**: `wbs-excel`

1. 祝日リストを定義
2. フェーズ・タスクを設計
3. WORKDAY関数で終了日自動計算
4. ガントチャート自動生成（曜日表示付き）
5. バージョン付きファイル名で保存

**出力**: `WBS_v1.0.xlsx`

### Phase 3: ワークプラン作成（スライド）

**Skill参照**: `workplan-slide`

1. WBSを読み込み
2. フェーズを5-6行に統合
3. 矢羽根を1行2-3本にまとめる
4. マイルストーンを追加
5. バージョン付きファイル名で保存

**出力**: `workplan_v1.0.pptx`

---

## 絶対に守るルール【重要】

### WBS作成時

| ルール | 理由 |
|--------|------|
| 祝日リストを最初に定義 | WORKDAY関数の正確性 |
| 曜日を表示 | 日付感覚の把握 |
| バッファを確保 | 最終報告日の2-3日前に完了設計 |
| WORKDAY関数を使う | 終了日の自動計算 |

### ワークプラン作成時

| ルール | 理由 |
|--------|------|
| WBSを先に読む | 期間・フェーズの正確な把握 |
| 5-6行に統合 | スライドの見やすさ |
| 短い矢羽根は統合 | 文字が読めるサイズ確保 |
| マイルストーンを追加 | 進捗の節目を明示 |
| 開発+テストは1行 | 行数削減 |

---

## 過去の失敗パターン（繰り返さない）

### WBS

| 失敗 | 正解 |
|------|------|
| バッファなし | 最終日2-3日前に完了設計 |
| 祝日漏れ | 年末年始・祝日リスト必須 |
| 曜日なし | ヘッダーに曜日表示 |

### ワークプラン

| 失敗 | 正解 |
|------|------|
| WBS詳細をそのまま転記 | フェーズ単位で統合 |
| 短い矢羽根を大量生成 | 短いタスクは統合 |
| 開発とテストを別行 | 「開発①」として統合 |
| マイルストーン少なすぎ | 開発開始、完了等を追加 |
| 「開発・改善①」 | 「開発①」（シンプルに） |
